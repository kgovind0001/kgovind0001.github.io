    Predicting accuracy of medical blogs on colon cancer using Llama2

    Introduction

    There are a number of medical blogs available that provide a lot of useful information about colon cancer. For example, Mayo Clinic, ESMO, and Moffitt provide detailed information about the symptoms and medical treatments available for colon cancer. However, it is crucial to analyze the quality of information available in such medical blogs and predict the accuracy to ensure reliability.

    Using AI models like Llama2, we can enhance the accuracy of the information provided in these blogs. By training Llama2 on verified medical data, it can cross-check the content of these blogs, identify inaccuracies, and ensure the reliability of the information. This approach helps users access accurate health information and make informed decisions.

    In this blog, we will explore how state of the art large language model, Llama2 can be utilized to assess and enhance the quality of information in medical blogs about colon cancer. Our focus is to take the blog URL as input and answer the following five (Yes/No) questions regarding the content of the blog:

    1. Is the definition of colon cancer provided in the blog?
    2. Does the blog inform about medical treatments for colon cancer?
    3. Are the precautions a patient might need to take when having colon cancer explained in the blog ?
    4. Does the blog mention early signs/symptoms of colon cancer that the patient may notice?
    5. Are relevant sources provided to get information or support when having colon cancer?

    Solution

    Step1: Extract information of the blog content.

    The first step is to collect information about blogs that provide valuable insights related to colon cancer. A total of 40 such blogs were found. Once the blog URLs were identified, it was observed that the Langchain URLLoader can be very useful for scraping the contents of these blogs. We can use the below code to scrape the information.

    
url = "https://www.mayoclinic.org/diseases-conditions/colon-cancer/symptoms-causes/syc-20353669"
loader = UnstructuredURLLoader(urls=[url], mode="elements",ssl_verify=False)
data = loader.load()

    The method extracts the information in an unstructed manner and need to be further formatted. It is important to note that the above method would extract only the content of the blogs, and some of the most important information might be available on other associated web pages. In such cases, it might be necessary to first identify these useful links and then scrape data from these pages as well. This can be done by first finding the associated hyperlinks on the page, which can be accomplished using playwright library from Langchain.

    # Extract text and the corresponding hyperlinks.

    Once the information has been extracted from the blogs, we can start moving towards using the LLM model to query the blog content and obtain answer to the questions.

    Step2: Retrive the most important content from the blog for a query.

    LLMs can reason about a wide range of topics, but their knowledge is limited to the public data available up to the point in time when they were trained. To build AI applications that can reason about private data or data introduced after a model's cutoff date, you need to augment the model's knowledge with the specific information it needs. This process of bringing the appropriate information and inserting it into the model prompt is known as Retrieval Augmented Generation (RAG).

    RAG is particularly important when dealing with blogs, where essential information required to answer a question may be located in a specific portion of the content. By using RAG, you can efficiently retrieve the most relevant pieces of information from a large blog, ensuring that the model has access to the necessary context to provide accurate and relevant responses.

    Retrieval Augmented Generation (RAG)

    First, the entire content of the blog is taken and divided into smaller, more manageable pieces called chunks. This is often done to improve the efficiency of processing and to ensure that each segment can be analyzed in a meaningful context. The size of the chunks can vary depending on the length of the blog posts and the quality of information you need.

    Once the blog is divided into chunks, the next step is to convert these text chunks into a format that can be understood which is done using embeddings. An embedding is a vector representation of text that captures the semantic meaning of the words and their relationships. We useall-MiniLM-L6 v2 model from the Sentence Transformers library, which is based on Microsoft's MiniLM model. It's designed to generate high-quality language representations (or embeddings) for sentences and paragraphs. When we use this model to process each text chunk, it converts the chunk into a vector (a list of numbers). These vectors are then stored in a database like Faiss, ChromoDB. Each vector represents a chunk of the blog, capturing its semantic essence in a numerical form. This vector database allows for the efficient comparison and retrieval of text based on semantic similarity rather than simple keyword matching.

    from langchain.text_splitter import RecursiveCharacterTextSplitter
from langchain.embeddings import HuggingFaceEmbeddings
from langchain.vectorstores import FAISS


text_splitter=RecursiveCharacterTextSplitter(chunk_size=300, chunk_overlap=20)
docs=text_splitter.split_documents(data)

embeddings=HuggingFaceEmbeddings(model_name='sentence-transformers/all-MiniLM-L6-v2')
db = FAISS.from_documents(docs, embeddings)

    To retrieve information from the blog, we can query it

    
results = db.max_marginal_relevance_search(query,k=1)

    Each query is also transformed into a vector using the same MiniLM-L6 v2 model. This ensures that the query and the blog chunks are represented in the same semantic space and can thus be compared directly. With the query now also as a vector, you can search the vector database for the chunks (now vectors) that are most semantically similar to the query vector. This is typically done using techniques like cosine similarity, where the angles between the query vector and the document vectors are calculated to find the closest matches.

    The chunks with the highest similarity scores are retrieved and presented as context for the query. This allows users to get precise information from the blog that is most relevant to their questions.

    Step3: Finetune Llama2 model.

    We can directly use the content obtained from query of the questions in the the Llama2 model to get the answer to the questions.

    
    def get_answer(question,context):
        template = """
Answer the following question based on the given context. Answer only based on the context truthfully without assuming any knowledge outside the context. 

QUESTION: {question} 

CONTEXT: {context}

Now answer the question based on the context. 
ANSWER:
           """
        prompt = PromptTemplate(template=template, 
                                input_variables=["question","context"])
        llm_chain = LLMChain(prompt=prompt, 
                             llm=llm)
        response = llm_chain.invoke(question=question,
                                 context=context)
        return response


question = "Is the definition of colon cancer explained in the text ?"
context = """

Early detection of colon cancer can improve treatment outcomes and reduce the risk of death from the disease.

    The response can be improved by first creating a summary of the context and then using that as context in template.

    Conclusion

    The evaluation of blogs can be performed using LLMs where the blog content is first used to query the most important content for a query. This filters the important relevant content to make the decision. Results can be improved by fine-tuning LLMs on the training data, which leads to more accurate and relevant responses.

    By specifically fine-tuning the LLaMA2 model on a dataset that includes high-quality blog content and relevant evaluation metrics, we can enhance the model's ability to assess the quality of blogs more effectively. The fine-tuning process involves:

    1. Data Collection: Gathering a diverse set of blog posts along with their evaluations based on predefined quality criteria.
    2. Training Data Generation: Using ChatGPT to generate additional training data and manually correcting it to ensure high quality.
    3. Fine-Tuning: Training the LLaMA2 model on this refined dataset to adapt it to the specific task of blog evaluation.

    Through this process, the fine-tuned LLaMA2 model can provide more precise assessments of blog content quality, aiding in the identification of valuable and reliable blog posts. This approach not only automates the evaluation process but also ensures consistency and scalability in assessing large volumes of blog content.
